# v1.1 — Comprehensive Pre-Submit TestFlight Checklist

> **This is the go-live gate for v1.1.** Per the hard rule ([memory: pre-submit TestFlight QA gate]),
> v1.1 is **not submittable for App Store review until this passes top-to-bottom on a real device.**
> It **supersedes** [PHASE_B_TESTFLIGHT_CHECKLIST.md](PHASE_B_TESTFLIGHT_CHECKLIST.md) for the
> full-version pass — that one validated only the Phase B *exit subset* (IAP + PDF). This covers the
> **entire v1.1 surface**, and especially the **~11 features that landed *after* that pass and have
> never run on a device**: mileage-log fields, custom expense categories, W-4 optimizer, safe-harbor,
> year-over-year, and the four PDF/on-screen follow-ons.

**Why device-only:** off-device is already green (tsc, 144 unit tests, 17-test Playwright, Maestro
flows authored). None of that exercises the **native paths** that break in the real world — the
priority of this pass. Run it as **one build** to respect the Codemagic budget; don't ship a second
build to catch a missed step. Check the box only when verified on the device.

---

## 0. Before the build

- [ ] **Sandbox tester** exists (App Store Connect → Users and Access → Sandbox → Testers) and you're
  signed into it on the device (Settings → App Store → Sandbox Account) — required to buy without a charge.
- [ ] **IAP review screenshot** attached to both products in ASC (needed for Apple to *approve* the
  IAP, not for testing) — ask Claude to generate a paywall screenshot if not done.
- [ ] Build is the **latest `v1.1`** (push first) at **version 1.1.0**; installed fresh via TestFlight.
- [ ] Test device is a real iPhone (not just the simulator) — Face ID/notifications/share/IAP are the point.

## 0.1 Seed a realistic persona (so premium screens are populated)

- [ ] Profile: real-sounding name; **a state WITH income tax** (e.g. NY/CA) so federal **and** state lines show.
- [ ] Turn **"I also have a W2 job" ON** with believable pay-stub numbers (required for the W-4 shot in F5).
- [ ] Log **6–8 entries** across 3–4 platforms, over the last couple of weeks, believable amounts; several
  with mileage + expenses; at least one with hours (so effective-hourly-rate shows).
- [ ] On a couple of trips, fill the **mileage-log** purpose/start/end and add a **custom expense** or two.
- [ ] Set "amount set aside so far" into the **green "on track"** band.

## A. Free core smoke (the v1.0 base, re-verified in the merged build)

- [ ] Onboarding / profile form completes; dashboard renders with the **set-aside gradient card** (SE + federal + state lines).
- [ ] **Log Earnings** — the **native date picker** opens, sets a date, and the entry saves; the set-aside number updates immediately.
- [ ] **Edit** an existing entry and **delete** one (native destructive confirm) — totals recompute correctly.
- [ ] **Year switcher** changes the active tax year; numbers scope to the selected year.
- [ ] **Dark mode** (Settings → Appearance) — all core screens readable, no clipped/again-white cards.
- [ ] Rotate / large Dynamic Type sanity (no critical clipping on the numeric screens).

## B. Free Phase A features

- [ ] **Show your math** — tap a breakdown line (federal / SE / state); the sheet opens and the math reconciles (AGI − std deduction = taxable; brackets sum to the row).
- [ ] **Tax-literacy glossary** — the "what these mean" pills expand plain-language definitions inline.
- [ ] **What-if simulator** — change hypothetical earnings; set-aside + effective hourly rate recompute live.
- [ ] **Platform comparison** — per-platform earnings + after-expense hourly rate render for the seeded platforms.
- [ ] **Share card (NATIVE — view-shot + share sheet):** tap share → the branded image renders and the **iOS share sheet** opens; AirDrop/Save-to-Photos produces a real image (this is the `react-native-view-shot` path Playwright can't test).
- [ ] **Rating prompt (NATIVE — StoreReview):** after the 5th entry / first catch-up, the OS rating dialog appears **once** and doesn't re-nag (may be rate-limited by iOS in sandbox — confirm it at least doesn't crash).

## C. Native trust / privacy

- [ ] **Biometric app lock** — enable in Settings; background + reopen → **Face ID / Touch ID** challenge gates entry; a failed/cancelled auth keeps data locked.
- [ ] **Quarterly reminder notification** — the app schedules the estimated-tax due-date reminder; the OS permission prompt appears and a scheduled local notification is registered (verify in the pending list / by fast-forwarding the device date if practical).
- [ ] **CSV export (native share)** — exports a CSV via the share sheet; open it and confirm columns incl. the **mileage-log** fields and the **custom-expenses** column.
- [ ] **Backup → Restore round-trip** — create a backup file, wipe/reinstall (or restore over), and confirm entries + profile come back intact (native file read/write).

## D. Premium gating from the FREE state (each entry point routes to the paywall)

_Before purchasing, confirm every gated surface locks correctly and routes to the paywall — several are
native "locked row → Alert → paywall" hops only Maestro exercises off-device:_

- [ ] Settings **"Tax Summary (PDF) · Premium"** → paywall (does not export).
- [ ] Add-Entry **"IRS mileage log"** locked Premium row → Alert → paywall; returns to the entry form on dismiss.
- [ ] Add-Entry **"Custom expense categories"** locked Premium row → Alert → paywall.
- [ ] Dashboard **"Expense breakdown · Premium"** card (shown when a deductible expense/mileage exists) → straight to paywall.
- [ ] Dashboard **"Skip quarterly payments · Premium"** (W-4) card (shown only with the W2 job on) → paywall.
- [ ] Dashboard **"Avoid the IRS penalty · Premium"** (safe-harbor) card → paywall.
- [ ] Dashboard **"Year-over-year insights · Premium"** card (shown once entries span 2+ years — see F7) → paywall.
- [ ] Paywall itself (Apple 3.1.2): both **RevenueCat prices** ($29.99/yr, $4.99/mo), auto-renewal disclosure, Terms/Privacy links, Subscribe, Restore — price is the most prominent element.

## E. Purchase → unlock → restore → offline

- [ ] **Subscribe (annual)** → native sandbox purchase sheet → confirm → paywall dismisses; app reflects **premium active** (locks gone, "· Premium" suffixes drop).
- [ ] Analytics funnel fired: `paywall_viewed → purchase_started → purchase_completed` (confirm in PostHog if convenient).
- [ ] **Restore** — delete + reinstall (or sign out/in) → **Restore purchases** → premium returns.
- [ ] **Offline entitlement** — airplane mode with an active sub → premium **stays unlocked** and exports still work.

## F. Premium features (now unlocked)

**F1 — PDF tax-ready export (NATIVE print → share):**
- [ ] With ≥1 entry this year, export → PDF renders, **system share sheet** opens, file named `tax-summary-<year>.pdf`.
- [ ] **Schedule C** numbers reconcile: Line 1 gross = earnings + tips; **Line 9** = mileage (miles × rate, per the note) + parking + tolls; Line 22 supplies; Line 25 phone; **Line 27 "Other expenses"** shows the **per-custom-category** sub-rows; Line 31 net = gross − expenses.
- [ ] **Mileage-log appendix** ("Schedule C Line 9 Substantiation") present: one row per business-mile trip (date · purpose · route · miles), oldest-first, total reconciles to Line 9; blank purpose falls back to the platform label, blank route to a dash; **no zero-mile rows**.
- [ ] **Safe-harbor section** present: both legs (90% current vs 100/110% prior) with the **binding leg marked**, required annual payment, W-2 withholding credited (row omitted if 0), estimated payments, **≈ per-quarter 1040-ES** figure — OR a plain no-penalty note when under the $1,000 floor.
- [ ] "Estimated amount to set aside" in the PDF **matches the dashboard**. Header (name, filing status, state/county, date) + disclaimer present.
- [ ] **Empty-year guard:** with no entries this year, export shows the "log at least one entry" alert, not an empty PDF.

**F2 — Mileage-log fields:** the purpose/start/end entered in 0.1 appear in the **CSV** (C) and the **PDF appendix** (F1).

**F3 — Custom expense categories:** the custom lines reduce **net SE profit** in the estimate, land on **Line 27** in the PDF, and show a **CSV** column.

**F4 — On-screen expense breakdown:** open the screen → Line 9 composition (mileage + parking/tolls), 22, 25, 27 with per-custom-category sub-rows, Line 28 total, and net profit — all matching the PDF; empty/hint states behave.

**F5 — W-4 optimizer (needs the W2 job on):** result screen shows the extra per-paycheck **Line 4(c)** withholding + a date-aware catch-up figure; sanity-check it's roughly (gig tax ÷ remaining paychecks).

**F6 — Safe-harbor / Form 2210:** enter last year's **filed federal tax** (prior-year input; a suggestion may prefill) → the prior-year leg binds when income jumped; the required amount and per-quarter figure are sensible; MFS/110% threshold behaves.

**F7 — Year-over-year (NATIVE date picker to seed a 2nd year):** log an entry dated **in the prior tax year** (drive the native picker back a year) so entries span 2 years → the dashboard YoY card appears → the screen shows the latest-vs-previous deltas (earnings/profit/hourly-rate colored by direction; expenses/miles/tax neutral), the all-years table, and — if a filed figure was entered in F6 — the **"Filed federal tax on record"** card (separate from the combined estimate, with its footnote). With only one year, it shows the friendly "come back next year" state instead.

## G. Financial-accuracy spot-checks (cross-cutting)

- [ ] The **set-aside number** is consistent everywhere it appears (dashboard, PDF, what-if base case).
- [ ] Changing filing status / state in the profile moves the numbers in the expected direction.
- [ ] A **zero-income / no-tax-owed** state renders sanely (no negative set-aside, no NaN) — the boundary case.

## H. Stability & observability

- [ ] No crashes across the full run; background/foreground and rotate without state loss.
- [ ] **Sentry** receives events (confirm the build reports in; optionally trigger a non-fatal to verify symbolication).
- [ ] **PostHog** shows the session + the premium-funnel events from E.

## I. Release / regression gate (off-device, confirm green before submitting)

- [ ] `tsc` clean · **144 unit tests** green · **17-test Playwright** e2e green.
- [ ] Maestro flows green on the Codemagic mac run (the native gating hops).
- [ ] Production/Release build succeeds (the TestFlight build itself is the proof).
